# Google スプレッドシート接続 — owner 向け 10分手順

この手順は、LINE ハーネスから自社の Google スプレッドシートへ安全に接続する準備です。Google の「サービス アカウント」を 1 つ作り、対象シートだけを編集者として共有します。

先に大切な点を 2 つだけ確認してください。

- ダウンロードする JSON は、パスワードと同じ秘密値です。チャット、メール、スクリーンショット、Git、作業ログへ貼らないでください。
- ブラウザへ表示するのはサービス アカウントのメールアドレスだけです。JSON キーや秘密鍵はブラウザへ送りません。
- 接続後は、フォームビルダーで選んだフォーム項目だけが設定した方向へ同期されます。最初は必ず架空の回答だけで確かめてください。

## 用意するもの（1分）

- Google Cloud を操作できる Google アカウント
- 接続確認用の空のスプレッドシート。最初は個人情報や本番データを入れないでください
- LINE ハーネスの owner 権限
- trusted host で Worker secret を登録できる担当者。ブラウザだけを使う owner は、手順 5 のコマンド実行だけ host 担当者へ渡してください

## 1. Google Sheets API を有効にする（2分）

1. [Google Cloud Console](https://console.cloud.google.com/) を開きます。
2. 画面上部のプロジェクト名を押し、専用プロジェクトを選びます。無ければ「新しいプロジェクト」で作ります。
3. 左上メニューから「API とサービス」→「ライブラリ」を開きます。
4. `Google Sheets API` を検索して開き、「有効にする」を押します。「管理」と表示されれば完了です。

スクリーンショット位置 1: `Google Sheets API` の詳細画面で「API が有効です」と分かる部分だけを撮ります。プロジェクト ID、課金情報、通知は写さないか、共有前に隠してください。

## 2. サービス アカウントを作る（2分）

1. Google Cloud の左上メニューから「IAM と管理」→「サービス アカウント」を開きます。
2. 「サービス アカウントを作成」を押します。
3. 名前に `line-harness-sheets` など用途が分かる名前を入れ、「作成して続行」を押します。
4. この接続ではシート側の共有権限を使うため、プロジェクトの役割は追加せず「続行」→「完了」で構いません。
5. 一覧に出たメールアドレスを、手順 4 で使えるよう安全なメモへ控えます。

スクリーンショット位置 2: サービス アカウント一覧の「名前」と「有効」が分かる行を撮ります。メールアドレスは外部共有する証跡では隠してください。

## 3. JSON キーを 1 回だけ発行する（1分）

1. 作ったサービス アカウントを開き、「キー」タブへ進みます。
2. 「鍵を追加」→「新しい鍵を作成」→「JSON」→「作成」を押します。
3. JSON ファイルが自動でダウンロードされます。中身を開いて画面共有したり、ファイルをチャットへ添付したりしないでください。

この画面ではスクリーンショットを撮らないでください。秘密鍵は再表示できません。紛失した場合は探し回らず、古い鍵を無効にして新しい鍵を作ります。

## 4. 確認用シートをサービス アカウントへ共有する（1分）

1. 個人情報を含まない空の確認用スプレッドシートを開きます。
2. 右上の「共有」を押し、手順 2 で控えたサービス アカウントのメールアドレスを入れます。
3. 権限を「編集者」にして共有します。双方向連携では読み取りと書き込みの両方が必要です。
4. ブラウザ上部の共有 URL をそのままコピーします。URL の一部を手で抜き出す必要はありません。
5. 画面下に実在するタブを 1 つ以上用意します。最初の確認では `Sheet1` のままでも構いません。

スクリーンショット位置 3: 共有ダイアログでサービス アカウントが「編集者」になった状態を撮ります。セル内容、スプレッドシート ID、メールアドレスは証跡へ出す前に隠してください。

## 5. JSON を Worker secret に登録する（2分・trusted host 担当）

査読済みコードを使う trusted host でリポジトリの `apps/worker` に移動します。コマンド行へ JSON 本文を書かず、Wrangler の入力待ちになってからファイルの中身全体を貼ります。

```bash
cd apps/worker
set +x
pnpm exec wrangler secret put GOOGLE_SERVICE_ACCOUNT_JSON --config wrangler.ks.toml
```

- 入力待ちで JSON 全体を貼り、確定します。コマンド履歴には秘密値が残りません。
- JSON のファイル名や内容をログへ出さないでください。
- 登録後はダウンロードした JSON を通常のごみ箱へ移し、ごみ箱も空にします。組織の秘密管理庫を使う決まりがある場合だけ、そこへ保管します。
- `wrangler.toml`、`.env`、ソースコードへ JSON を書かないでください。

## 6. フォームビルダーで登録して接続テストする（2分）

1. owner で LINE ハーネスへログインし、左上で対象の LINE アカウントを選びます。
2. フォーム一覧から接続したい自前フォームのフォームビルダーを開き、「回答後の動き」へ進みます。
3. 「自前シート連携」に表示されたサービス アカウントのメールアドレスをコピーし、手順 4 のシートがそのアドレスへ共有済みか確認します。
4. 手順 4 でコピーした共有 URL を貼り、「接続を確認」を押します。
5. 読み込まれた実在タブから対応先を選び、「同期するフォーム項目」のチェックを確認します。通常の同期方向は「双方向」です。
6. 「シート連携を保存」を押します。難しい ID を入力する欄はありません。
7. `/settings/sheets` の確認ページを開き、フォーム名・接続状態・最終同期・エラーを確認します。必要な場合だけ「接続テスト」を 1 回押します。

スクリーンショット位置 4: フォームビルダーの「自前シート連携」で接続成功と実在タブが分かる部分を撮ります。共有 URL、回答内容、LINE アカウント名は外部へ出す前に隠してください。

スクリーンショット位置 5: `/settings/sheets` の確認ページで、フォーム名と接続状態が分かる部分を撮ります。内部 ID やシート内容は記録しません。

## うまく接続できないとき（1〜3分）

- 「接続設定が未完了」: `GOOGLE_SERVICE_ACCOUNT_JSON` が対象 Worker に入っているか、host 担当者が確認します。
- 「スプレッドシートの共有設定に上のアドレスを追加してください」: 画面上部のサービス アカウントのメールアドレスを、対象シートの共有先へ追加します。双方向なら「編集者」にします。
- タブが出ない: Google Sheets API が有効か、貼った共有 URL が正しいシートかを確認し、もう一度「接続を確認」を押します。
- owner 以外で開けない: 正常です。この設定は個人情報の同期先を決めるため owner 限定です。
- JSON を誤って共有した: 直ちに Google Cloud のサービス アカウント→「キー」でその鍵を無効・削除し、新しい JSON キーを作って Worker secret を入れ替えます。

保存後は、検証用フォームから架空の回答を 1 件だけ送り、選んだタブへ選択済みの項目だけが反映されることを確認してください。細かな状態確認や手動同期は `/settings/sheets` を使い、接続先や項目を変えるときは各フォームの「回答後の動き」へ戻ります。
