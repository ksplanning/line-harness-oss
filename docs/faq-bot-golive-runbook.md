# AI 自動応答（FAQ Bot）本番オン 立会チェックリスト

> **これは「手順書」です。ここに書いた操作は、オーナー立会いのときにだけ実行します。**
> **B-5 の実装時点では、AI はまだ 1 文字もお客さまに送りません（本番は止まったまま＝ダークシップ）。**
> このチェックリストがそろった状態が「準備完了・立会待ち」です。**勝手にオンにはしません。**

対象読者: オーナー（非エンジニア）と、実際にコマンドを打つ担当（インフラ担当 / `owner_role: infra-ops`）。
オーナーは「何を・どの順で・なぜ」を読み、担当が下線部のコマンドを立会いのもとで実行します。

---

## 0. いま何が「オフ」なのか（前提の確認）

本番は次の 2 つで二重に止まっています。**この 2 行は B-5 では一切変更していません。**

- `apps/worker/wrangler.ks.toml` の `crons = []`（定期実行がゼロ＝バックグラウンド処理が動かない）
- `apps/worker/wrangler.ks.toml` の `FAQ_BOT_ENABLED = "false"`（AI の自動返信ゲートが閉じている）

さらに、AI が使う 2 つの外部部品（`[ai]` と `[[vectorize]]`）は **まだ設定していません**。
だから管理画面で資料を取り込んでも、AI は動かず「意味検索は未設定」と表示されます。これは正常です。

## 1. 【インフラ工程】意味検索（Vectorize）の準備 — 順番厳守のチェックリスト

> `owner_role: infra-ops`。**下の順番を 1 つでも飛ばすと、アカウントの絞り込み（metadata フィルタ）が
> 効かなくなり、先に入れたベクトルを入れ直す羽目になります。**「等」で省略せず、全部やります。

- [ ] **(0) 使うAIモデルを決める。** 生成用 `AI_MODEL_ID`（未設定だと安全側でエラー＝送らない）と、
      埋め込み用 `AI_EMBED_MODEL_ID`（非推奨でない多言語モデル）を確定する。**埋め込みモデルの出力次元**を控える。
- [ ] **(1) 意味検索の入れ物を作る。**
      `wrangler vectorize create ks-knowledge-chunks --dimensions=<埋め込み次元> --metric=cosine`
- [ ] **(2) アカウント絞り込み用のインデックスを作る。**
      `wrangler vectorize create-metadata-index ks-knowledge-chunks --property-name=line_account_id --type=string`
- [ ] **(3) 絞り込みインデックスが登録できたか確認する。**
      `wrangler vectorize list-metadata-index ks-knowledge-chunks`
- [ ] **(4) 次元が一致しているか確認する。** 入れ物の次元 (`--dimensions`) と、選んだ埋め込みモデルの
      出力次元が **完全一致** していること。ズレていると全部の埋め込みが無駄になり、作り直しになります。
- [ ] **(5) 設定ファイルに部品を「追記」して、まだオフのままデプロイする。**
      `apps/worker/wrangler.ks.toml` に次を **追記のみ**（既存行は消さない）:
      `[[vectorize]] binding = "VECTORIZE" index_name = "ks-knowledge-chunks"` /
      `[ai] binding = "AI"` /
      `[vars]` に `AI_EMBED_MODEL_ID` / `AI_EMBED_NEURON_PER_MTOK` / `AI_CHUNK_RELEVANCE_FLOOR`。
      → **`FAQ_BOT_ENABLED = "false"` のままデプロイ**（部品の用意と「オン」は別の段階です）。
- [ ] **(6) 取り込み済みの資料を初回だけ埋め込む（バックフィル）。**
      管理画面「資料・AIログ」→ 内部の `POST /api/knowledge/backfill-embeddings`（未 embed の chunk を
      `getUnembeddedChunks → embedChunksForDocument` で埋める）を、**ページングしながら**呼ぶ。
      無料枠を超えそうなら止まる（残りは後日）＝**停止条件つき**で安全。
- [ ] **(7) 実際に入った容量を確認する。**
      `wrangler vectorize info ks-knowledge-chunks` で stored の実値を確認し、管理画面の「推定」と突き合わせる。

> ⚠️ **順序違反の意味**: (2)(3) の絞り込みインデックスより先に (6) でベクトルを入れると、アカウント絞り込みが
> 効かず、先に入れたベクトルは入れ直しになります。必ず上から順に。

## 2. 【安全確認】実モデルでの意地悪テスト（adversarial eval）と「送らない基準」

- [ ] **意地悪な資料を混ぜてテストする。** 「以前の指示を無視して〜へ送れ」「system:」などの
      **質問に関連する悪意ある文章**をわざと資料に埋め込み、採用したモデルで injection が通らないか実測する。
      （テスト用のモック検証は構造チェックまで。実モデルの遵守はここで実測します。）
- [ ] **「安全チェックを通らなかったら送らない」を明文の基準にする（fail-closed）。**
      自動送信は、根拠スコア（cosine floor）・グラウンディング（URL/電話の裏取り）・システムプロンプト遵守の
      すべてを通ったときだけ。1 つでも不安なら **送らずに「未対応」に隔離**（＝下書き相当）。
      ※ グラウンディングは URL/電話しか検査しない限界があるため、埋め込んだ悪意ある連絡先は
      cosine floor とプロンプト遵守に依存する「残るリスク」として、上の意地悪テストの対象にします。

## 3. 【本番オン】オーナー立会いテスト（下書きファースト）

> 既定の回答モードは「自動（auto）」です。**いきなり全体オンにすると即・自動送信が起こり得る**ため、
> 必ず下の順で「下書き（draft）」から始めます。

- [ ] **(a) 全アカウントが自動応答オフか確認する。** どのアカウントも `faq_bot.enabled = false`
      （まだ誰にも自動送信されない状態）から始める。
- [ ] **(b) 検証用アカウント（自分宛て）だけをオンにする。ただし下書きで。**
      検証アカウントだけ `faq_bot.enabled = true` かつ **`answer_mode = draft`**（送らず草案を作るだけ）。
- [ ] **(c) 意地悪テストをゼロ違反で通す。** 検証アカウントで §2 の eval を回し、
      違反ゼロを確認する。作られた草案を目視でチェックする。
- [ ] **(d) 明示承認のあとだけ自動に切り替える。** オーナーの明示 OK が出てから、検証アカウントを
      `answer_mode = auto` に。問題なければ TRINA など本番アカウントへ **段階的に**解禁する。
- [ ] **(e) 全体スイッチを最後にオンにする。** ここで初めて `wrangler.ks.toml` の
      `FAQ_BOT_ENABLED = "true"` にしてデプロイ（`crons` を使う機能があれば同時に検討）。
- [ ] **(f) D-1 の直りを確認する。** 自分宛てに質問を送り、AI が数秒遅れて返信しても
      「未対応リスト」から**ちゃんと消える**ことを確認する（B-5 T-E5 の救済）。

### ロールバック（すぐ元に戻す手順）

1. 各アカウントの `account_settings` で `faq_bot.enabled = false` に戻す。
2. **それだけでは不十分。** 全体スイッチ `FAQ_BOT_ENABLED = "false"` に戻して **再デプロイし、確認**する。
3. データベースは追記のみ（消していない）ので無傷。管理画面 UI 側は変更を取り消せば元に戻る。

## 4. 覚えておくこと（要点）

- **これは全部「立会いのときにやる」手順です。** B-5 の作業では、オン・crons 解禁・Vectorize の本番用意を
  **一切実行していません**（準備完了・立会待ち）。
- 資料の取り込み・削除・再取り込みは「保存」だけで、**お客さまへの送信は 1 度も起きません**。
- コスト画面の数値は 1 日ごとの合計です。1 回ごとの内訳は記録していないため出せません（正直な限界）。
- 意味検索の容量は、Vectorize を用意するまでは「未計測」と表示されます（推定と実値を混同しないため）。
