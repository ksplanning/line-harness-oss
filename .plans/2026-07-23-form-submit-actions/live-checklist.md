# form-submit-actions — host live checklist

この確認は host closer が、実際のブラウザ・Worker・D1 を使って行う。

## 事前準備

- [ ] 検証用の友だちを 1 人用意し、現在のタグとカスタムフィールド値を記録する
- [ ] 検証用タグを 2 個作る（例: `フォーム前`、`フォーム完了`）
- [ ] 検証用カスタムフィールドを 1 個作る（例: `申込状態`）
- [ ] 高度フォームを 1 個用意し、検証用の友だちとして回答できる公開 URL を確認する

## 設定の往復

- [ ] ビルダーの「送信後にやること」で、次の順に追加する
  1. `フォーム前` タグを付ける
  2. `フォーム前` タグを外す
  3. `フォーム完了` タグを付ける
  4. `申込状態` を `確認済み` にする
  5. `申込状態` を空欄にする
  6. `申込状態` を `完了` にする
- [ ] 上下ボタンで順番を入れ替え、不要な項目を 1 個追加してから削除できる
- [ ] 保存後にページを再読み込みし、種類・対象・値・順番がすべて同じである

## 友だちに紐付く回答

- [ ] 検証用の友だちとしてフォームを 1 回送信し、完了画面まで成功する
- [ ] 友だち詳細で `フォーム前` が付いていない
- [ ] 友だち詳細で `フォーム完了` が 1 個だけ付いている
- [ ] `申込状態` が最終値の `完了` になっている
- [ ] 同じ webhook を再送できる環境では再送し、タグが重複せず最終状態も変わらない
- [ ] 既存の送信後シナリオを設定した場合、その起動と LINE 送信回数が変更前と同じである

## 匿名回答と失敗時の安全確認

- [ ] 友だち情報を付けない公開 URL から送信し、回答自体は成功する
- [ ] どの友だちのタグ・カスタムフィールドも変わらない
- [ ] Worker ログに `[form-submit-action]`、`status:"skipped"`、`reason:"friend_not_linked"` があり、氏名・LINE user ID・設定値が含まれない
- [ ] 削除済みのタグまたはカスタムフィールドを含む設定でも回答自体は成功し、失敗した項目より後の有効な項目が実行される

## 後方互換と回帰

- [ ] 旧 `onSubmitTagId` だけを持つ既存フォームを送信し、そのタグが従来どおり 1 個付く
- [ ] 「送信後にやること」が未設定のフォームを送信し、回答・完了画面・通知が変更前と同じである
