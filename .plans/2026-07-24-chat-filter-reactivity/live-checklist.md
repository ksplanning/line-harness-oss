# chat-filter-reactivity — host live checklist

対象: `lane/sola/chat-filter-reactivity` の査読済み revision を host へ反映した後に実測する。

## 完了・絞り込みの即時反映

- [ ] `/chats` を開き、「未対応のみ」に該当するテスト用チャットを選ぶ。
- [ ] 「対応操作」→「解決済にする」を押し、画面遷移・再読み込み・30秒 poll を待たずに左一覧から消える。
- [ ] 「未読」絞り込みでも、解決済み操作後に対象行がその場で消える。
- [ ] 「対応中」絞り込みでも、解決済み操作後に対象行がその場で消える。
- [ ] Network で status 更新 API が成功し、LINE 送信 API (`/send`) が 0 件である。

## 開封・未読に戻す・失敗時 rollback

- [ ] 「未読」絞り込みで未読チャットを開くと、既存の開封 API 成功後に対象行がその場で消え、右詳細は「対応中」になる。
- [ ] 右詳細の「対応操作」→「未読に戻す」を押すと、対象行が「未読」一覧へその場で戻る。
- [ ] DevTools の offline 等で status 更新 API を失敗させた場合、楽観的に外れた行が一覧へ戻り、「ステータスの更新に失敗しました。」が表示される。

## サーバー真実との往復

- [ ] 成功操作後にブラウザを再読み込みし、操作直後と同じ絞り込み結果になる。
- [ ] migration が無く、確認中の LINE メッセージ送信が 0 件である。

## owner 確認
