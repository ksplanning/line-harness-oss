# spec: edit-branch-editability

正本の done conditions は、発注元の `tasks.md` に記載された D-1〜D-6 とする。

## 非表示になった既存回答の扱い

編集時に分岐項目を変更して項目が非表示になっても、その項目の既存回答は
`answers_json` から削除せず保持する。非表示項目の入力コントロールは無効化し、
ブラウザの送信対象および今回の入力検証対象から除外する。

この方針により、一時的な分岐変更によるデータ消失を防ぐ。初期表示時点で
非表示の回答値は HTML へ埋め込まず、非表示項目へ偽造 POST された値も
採用しない。保存済みの値は、項目が表示されて正規に編集されるまで維持する。

## 分岐編集の権限制御

- `allow_branch_edit` の既定値は `0`（変更不可）とする。
- `0` のとき、ロジックの `sourceFieldId` および `conditions[].sourceFieldId` に
  現れる回答項目を読み取り専用で表示し、POST された変更もサーバ側で拒否する。
- `1` のときだけ分岐元項目を通常の編集項目として扱い、共有 client 評価器で
  表示対象を入力のたびに再評価する。
- 署名・ファイル等、従来から読み取り専用の型は設定値にかかわらず編集不可とする。

## 管理画面と保存先

- `allow_branch_edit` は `formaloo_forms` の additive 列に保存し、既定 `0` とする。
- 回答後編集の設定は internal / Formaloo の両配信方式で肯定表現にする。
- 分岐編集トグルは `/ife/` を使う internal 配信でだけ表示し、回答後編集が
  OFF の間は操作不可にする。保存済み値自体は保持する。
- `allow_branch_edit` は外部 Formaloo API payload へ送らず、harness 内だけで管理する。
