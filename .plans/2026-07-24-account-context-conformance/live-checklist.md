# account-context-conformance — host live checklist

実測担当: host closer / owner

## 前提

- LINE アカウント A・B を用意し、メール差出人またはスタッフ通知先を別内容にしておく。
- 操作対象は左上の「アカウント切替」で選択したアカウントだけとする。
- この確認では「テスト送信」「一斉送信」を押さない。配信作成を確認する場合も下書き保存までにする。

## 左上切替への追随

- [ ] 左上で A を選び `/settings` を開く。ページ内に別の「設定する LINE アカウント」が無く、A のメール差出人が表示される。
- [ ] 「スタッフ通知」を開き、A の通知先だけが表示される。
- [ ] A の安全な設定値を変更して保存し、同じ画面を開き直して A に保存された値が再取得される。
- [ ] 左上で B へ切り替え、B のメール差出人・スタッフ通知先へ即座に切り替わり、A の値が混ざらない。
- [ ] 左上で A へ戻し、先ほど A に保存した値が再表示される。
- [ ] `/accounts` の各カードに個別の「メール差出人」ボタンが無く、上部の「通知設定を開く」から `/settings` へ移動できる。
- [ ] `/settings/formaloo-workspaces` では左上で選択中の1アカウントだけが既定ワークスペース設定の対象になり、A/B 切替で対象が切り替わる。

## ドメイン認証確認

- [ ] Resend ドメイン登録直後で状態が `not_started` のアカウントを選ぶ。
- [ ] 登録済みドメイン ID がある場合、「認証状態を確認」が有効で、状態表示が「認証待ち」になる。
- [ ] 「認証状態を確認」を押すと認証チェックが走り、結果の状態が再表示される。

## 一斉配信

- [ ] 左上で A を選び `/broadcasts` を開くと、A の配信一覧だけが表示される。
- [ ] 「単アカ配信」「複アカ重複除外」の独自タブと、新規配信内の「配信先アカウント」選択が無い。
- [ ] 新規配信の下書き保存先と下書きテスト送信先の表示が A に固定される（実際のテスト送信は行わない）。
- [ ] 左上で B へ切り替えると一覧・新規下書きの対象が B に切り替わり、A の値が残らない。

## owner 確認結果

- [ ] 上記を host で実測した。
- [ ] アカウント間の設定混線なし。
- [ ] LINE 送信ゼロ。
