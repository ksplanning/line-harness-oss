# settings-nav-page — host live checklist

実測担当: host closer（owner 確認項目）

## 事前条件

- owner 権限で host の管理画面へログインする。
- LINE アカウントが複数ある環境では、表示名の異なる2アカウントを確認対象にする。
- この確認では「テスト送信」を押さない。LINE 送信は行わない。

## 実測手順

1. 左メニューの「設定」見出しを開き、その中の「設定」を押す。
   - 期待: タイトルが「設定」のページへ移動し、「設定する LINE アカウント」が見える。
2. 確認する LINE アカウントを選び、「メール差出人」を押す。
   - 期待: 「通知メールの差出人」と「独自ドメインの認証」が見える。
   - 期待: DNS 設定の手順と「ドメインを登録してDNS設定を表示」が見える。
   - 期待: スタッフ通知の「通知チャネル」は同時に表示されない。
3. 「スタッフ通知」を押す。
   - 期待: 「登録済みの通知先」と「通知先を追加」が見える。
   - 期待: 「通知チャネル」で Chatwork と LINE を選べる。
   - 期待: メール差出人の DNS 設定は同時に表示されない。
4. 複数アカウントがある場合は別の LINE アカウントへ切り替え、手順2〜3をもう一度行う。
   - 期待: 選んだアカウントの設定として両パネルが開く。

## owner 確認欄

- [ ] 左メニューから迷わず設定ページへ到達できた。
- [ ] メール差出人は DNS 設定を含む画面だと判断できた。
- [ ] スタッフ通知は Chatwork / LINE の通知先画面だと判断できた。
- [ ] 2つの設定内容を取り違えなかった。
