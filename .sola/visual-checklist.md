# chat-history-popup — desk / host 実レンダー確認手順

この lane では dev server のポート公開ができないため、実レンダーは desk / host で行う。確認中は本番データを変更せず、送信・メモ保存・ステータス変更は押さない。既存のローカル fixture / mock データを使う。

## 起動

リポジトリルートで、ローカル Worker と Web を別ターミナルから起動する。

```bash
pnpm --filter worker dev
```

```bash
NEXT_PUBLIC_API_URL=http://127.0.0.1:8787 pnpm --filter web dev
```

ブラウザで `http://127.0.0.1:3001/chats` を開き、メッセージ履歴がある友だちを1人選ぶ。`/friends` から確認する場合は友だち行を押し、`/chats?friend=<friend-id>` へ遷移することも確認する。

## desktop（1440 × 900）

1. 右側の LINE 風チャット履歴が従来どおり表示され、送信欄・メモ・対応ステータス操作が元の狭い窓に残っていることを確認する。
2. ヘッダーのアイコン付き「拡大表示」ボタンが、説明を読まなくても5秒以内に見つかるか確認する。マウスポインターが変わり、hover で `クリックしてチャット履歴を大きく表示` が出ることも確認する。
3. 「拡大表示」を押し、背景が暗くなり、ダイアログが viewport の約90%（上下左右に約5%の余白）で開くことを確認する。
4. ダイアログ見出しが `<友だち名>のチャット履歴` で、狭い窓と同じ最初・最後のメッセージ、日付、時刻、画像 / Flex / スタンプが表示されることを確認する。
5. 開いた直後のスクロール位置が最新メッセージであることを確認する。上へスクロールして過去履歴を読め、閉じて再度開くと最新位置に戻ることも確認する。
6. 次の3経路を1回ずつ試し、毎回元の狭い窓と送信欄がそのまま残ることを確認する。
   - 右上の `×`
   - 暗い背景
   - `Esc`
7. キーボードの `Tab` で「拡大表示」へ移動して `Enter` で開けること、開いた直後は `×` にフォーカスすること、閉じると「拡大表示」へフォーカスが戻ることを確認する。

## mobile（390 × 844）

1. DevTools の viewport を `390 × 844` にする。
2. 個別チャットを開き、「拡大表示」が文字付き・44px 以上の押しやすいボタンとして見えることを確認する。
3. ボタンを押し、ダイアログが余白なしの全画面になり、見出しと `×` が画面内に収まることを確認する。
4. 横スクロールや切れた本文がなく、縦スクロールで過去から最新まで読めることを確認する。
5. `×`、背景相当の領域（desktop で確認）、`Esc`（キーボード接続時）で閉じられ、元のチャット画面に戻ることを確認する。

## おばあちゃん目線 / 両テナント

- 初見の人に操作説明をせず、「履歴を大きく見るにはどこを押すと思う？」と尋ねる。「拡大表示」を5秒以内に指せれば PASS。アイコンだけを手掛かりにしない。
- `×` が右上に常時見え、閉じ方を迷わないことを確認する。
- account switcher で line-harness-ks と piecemaker の各テナントを選び、同じ手順を繰り返す。友だち名・メッセージ・account ID が固定値になっていないことを確認する。
- desktop と mobile のスクリーンショットを各1枚、さらに `×` / 背景 / Esc の結果を確認メモとして desk のレビュー証跡へ添付する。
