# friend-fields-global-schema — 変更概要

「一回作れば全員の個人情報欄に同じ項目が出ます。全員分の入力は不要・未入力は 未 と表示」

テナント単位の「友だち項目定義」を追加しました。項目名・既定値・表示順・有効状態を一度設定すると、値をまだ持たない友だちにも同じ項目と既定値が表示され、友だちごとの値はこれまでどおり編集できます。定義外の個別項目と内部 marker の扱いも従来どおりです。

フォームの row-status mapping では定義済み項目を候補から選べ、従来の自由入力も残ります。配信出し分けでは、友だちに明示値がなければ有効な定義の既定値を使います。migration は項目定義テーブルを加えるだけで、既存の `friend.metadata` は変換・移行しません。

# row-status-friend-sync — 変更概要

シートで入金済みにすると、友だちの個人情報欄の入金確認も済になります。

フォームごとに「Formaloo の field slug / alias → 個人情報の項目名」を設定できます。未設定のフォームはこれまでどおり何もしません。署名済み `fr_id` で本人と確認できた最新 row だけを反映し、値が同じなら DB を書きません。Formaloo 側を空にした場合は個人情報側も空にします。反映元と最終更新時刻は内部 metadata に残し、個人情報画面には内部 marker を出しません。

手動編集との関係は、mapping 対象の項目だけ Formaloo が正です。手動で別の値にしても次の reconcile で Formaloo 値へ戻ります。mapping していない手動項目は残ります。

制約は 2 点あります。シートから Formaloo へ送る sidebar Sync は手動です。また本件の reconcile は管理画面の回答データまたは統計を開いた時に発火し、cron には未配線です。回答後編集を有効にした `/fo` では targeted pull でも同じ反映が走ります。緊急停止 flag は既存の friend_id 復元 / prefill も同時に止めます。

# richmenu-conditional-rules — 変更概要

友だちのタグや個人情報に合わせて、見せるリッチメニューを自動で切り替えられるようにしました。

リッチメニュー管理の「表示条件ルール」で、条件、表示するメニュー、優先度、有効・停止をいくつでも登録できます。複数の条件に合うときは優先度の数字が大きいものが勝ち、同じ数字なら先に作ったルール、その後は ID 順で決まります。画面には候補順位を表示するため、どのルールが勝つかを一覧で確認できます。

タグ名・タグの付け外し・個人情報・フォーム reconcile の反映で対象の友だちだけを再判定します。同じメニューのままなら LINE を呼ばず、どの条件にも合わなくなったら個別指定を外して、これまでの「全員のデフォルト」へ戻します。LINE が一時的に失敗した場合は友だち情報の更新を失敗させず、後で再試行できる状態を残します。

ルールを大きく変えた場合は「既存の友だちへ再適用」を押します。5 分ごとに最大 20 人ずつ進み、進捗と適用・変更なし・失敗の人数が見えます。連打はできません。ルールを一件も作っていないアカウントでは、従来の友だち追加とデフォルトメニューの動きは変わりません。

# admin-ai-chat-phase1 — 変更概要

管理画面で「今週の回答の傾向は？」と聞くと AI が答えます。

高機能フォームを選び、ふだんの日本語で質問すると、Formaloo 純正AIの回答をチャット風に表示します。質問・回答・
対象フォーム・分析番号・クレジット使用状態は履歴に残り、後から開き直せます。質問例も画面に置きました。

AI分析は費用が動くため既定ではOFFです。同じフォームの分析中は再送できず、テナントごとの日次上限もDBで守ります。
利用枠不足や時間切れは、成功したふりをせず日本語で理由を表示します。

sandbox ではクレジット保護のため実射していません。公式資料に未掲載のPOST本文と接続先aliasをhostで確認してから、
実射は host 工程で 1 回検証し、直後にflagを再びOFFへ戻します。

# richmenu-rule-schedule — 変更概要

キャンペーン期間だけ特別メニュー、が自動でできます。期間が明けたら自動で元に戻ります。

表示条件ルールに、日本時間の「いつから」「いつまで」を追加しました。開始だけ、開始と終了、終了だけを指定でき、両方を空欄にすれば今までどおり無期限です。期間はタグやカスタム項目の条件と一緒に判定されるため、期間外の高優先度ルールが勝つことはありません。一覧では「今有効」「開始前」「終了済み」「無期限」が見えます。

時間が進んで開始・終了をまたいだ時は、15 分ごとの確認で対象の友だちを再評価します。LINE の負荷を抑えるため、切替は既存の仕組みで 5 分ごとに最大 20 人ずつ進みます。同じメニューのままなら LINE を呼びません。期間が終わった後に次の条件へ合わなければ、個別指定を外して「全員のデフォルト」へ戻します。期間を設定していない既存ルールの判定結果は変わりません。
