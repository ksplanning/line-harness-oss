# internal-form-image-1mb-fix — 変更概要

自前フォームで「画像は10MBまで」と出るのに、1MBを超える画像が保存できないように見える問題を直しました。

調べると、フォーム内に置く画像はすでに正しく R2 へ逃がせていました。一方、デザイン画面のロゴと背景画像は、自前フォームの保存処理につながっておらず、選んだ画像そのものが保存から抜け落ちていました。今回、ロゴ・背景・フォーム内画像の3か所を同じ保存処理へそろえ、2MBの画像でも、まず R2 に置いてから小さい URL だけをDBへ保存するようにしました。保存後のフォーム定義には `data:image` を残しません。

画像を R2 に保存できなかったときは、成功したふりや分かりにくい500ではなく、「画像の保存に失敗しました (サイズ/形式)」という短い400エラーを返し、フォーム定義を変更しません。

Formaloo 用の保存 route、保護対象4ファイル、DB migration は変更していません。本番3フォーム、外部 Formaloo、LINE送信にも触れていません。sandbox では2MB画像の全3か所、R2失敗、D1の1MB binding guard、`data:image` 残存ゼロを自動テストし、worker全2,898テスト、型検査、Worker build、Webの64ページ static exportまで通しています。

各入力欄には、補足説明とは別にプレースホルダーを設定できます。選択項目の最初の選択肢、1行・複数行の最小/最大文字数、入力中の残り文字数も設定・表示できます。計算結果はブラウザの値を信用せず、送信時にサーバーでも計算し直します。

日時・国・日本の住所パーツは「自前フォーム」を選んだときだけ出ます。Formaloo 用フォームの作成・同期経路は変更せず、保護対象 4 ファイルも触っていません。DB migration も不要でした。本番 3 フォームと外部 Formaloo への実射は行っていません。

sandbox では worker 2,739件、web 1,326件、shared 477件、db 499件の全テストが通っています。実ブラウザでの最終 1 周は、査読後にこのファイルと同じ `.sola/live-checklist.md` の先頭の手順で行います。

# friend-fields-global-schema — 変更概要

「一回作れば全員の個人情報欄に同じ項目が出ます。全員分の入力は不要・未入力は 未 と表示」

テナント単位の「友だち項目定義」を追加しました。項目名・既定値・表示順・有効状態を一度設定すると、値をまだ持たない友だちにも同じ項目と既定値が表示され、友だちごとの値はこれまでどおり編集できます。定義外の個別項目と内部 marker の扱いも従来どおりです。

# row-status-friend-sync — 変更概要

シートで入金済みにすると、友だちの個人情報欄の入金確認も済になります。

フォームごとに「Formaloo の field slug / alias → 個人情報の項目名」を設定できます。未設定のフォームはこれまでどおり何もしません。署名済み `fr_id` で本人と確認できた最新 row だけを反映し、値が同じなら DB を書きません。Formaloo 側を空にした場合は個人情報側も空にします。反映元と最終更新時刻は内部 metadata に残し、個人情報画面には内部 marker を出しません。

手動編集との関係は、mapping 対象の項目だけ Formaloo が正です。手動で別の値にしても次の reconcile で Formaloo 値へ戻ります。mapping していない手動項目は残ります。

制約は 2 点あります。シートから Formaloo へ送る sidebar Sync は手動です。また本件の reconcile は管理画面の回答データまたは統計を開いた時に発火し、cron には未配線です。回答後編集を有効にした `/fo` では targeted pull でも同じ反映が走ります。緊急停止 flag は既存の friend_id 復元 / prefill も同時に止めます。

# richmenu-conditional-rules — 変更概要

友だちのタグや個人情報に合わせて、見せるリッチメニューを自動で切り替えられるようにしました。

リッチメニュー管理の「表示条件ルール」で、条件、表示するメニュー、優先度、有効・停止をいくつでも登録できます。複数の条件に合うときは優先度の数字が大きいものが勝ち、同じ数字なら先に作ったルール、その後は ID 順で決まります。画面には候補順位を表示するため、どのルールが勝つかを一覧で確認できます。

タグ名・タグの付け外し・個人情報・フォーム reconcile の反映で対象の友だちだけを再判定します。同じメニューのままなら LINE を呼ばず、どの条件にも合わなくなったら個別指定を外して、これまでの「全員のデフォルト」へ戻します。LINE が一時的に失敗した場合は友だち情報の更新を失敗させず、後で再試行できる状態を残します。

ルールを大きく変えた場合は「既存の友だちへ再適用」を押します。5 分ごとに最大 20 人ずつ進み、進捗と適用・変更なし・失敗の人数が見えます。連打はできません。ルールを一件も作っていないアカウントでは、従来の友だち追加とデフォルトメニューの動きは変わりません。

# richmenu-rule-schedule — 変更概要

キャンペーン期間だけ特別メニュー、が自動でできます。期間が明けたら自動で元に戻ります。

表示条件ルールに、日本時間の「いつから」「いつまで」を追加しました。開始だけ、開始と終了、終了だけを指定でき、両方を空欄にすれば今までどおり無期限です。期間はタグやカスタム項目の条件と一緒に判定されるため、期間外の高優先度ルールが勝つことはありません。一覧では「今有効」「開始前」「終了済み」「無期限」が見えます。

時間が進んで開始・終了をまたいだ時は、15 分ごとの確認で対象の友だちを再評価します。LINE の負荷を抑えるため、切替は既存の仕組みで 5 分ごとに最大 20 人ずつ進みます。同じメニューのままなら LINE を呼びません。期間が終わった後に次の条件へ合わなければ、個別指定を外して「全員のデフォルト」へ戻します。期間を設定していない既存ルールの判定結果は変わりません。

# ai-chat-rewire — 変更概要

「AIチャットの頭脳を Cloudflare 無料枠に切替。OpenAI 鍵を入れれば強い頭脳にも切替可能」

# ai-chat-verified-fix — 変更概要

# selfform-w4-sheets-foundation — 変更概要

Google スプレッドシートを LINE ハーネスへ直接つなぐ土台ができました。

owner はフォームごとにスプレッドシート ID、シート名、同期方向を登録できます。「接続テスト」を押すと、Google のサービスアカウントを使って先頭セルを 1 回だけ読み、共有設定が正しいかを確認します。秘密の JSON 鍵は画面や DB へ保存せず、Worker secret だけに置きます。

大切な制約として、まだ自動同期は始まりません。今回の範囲は接続、設定、疎通確認、同期台帳までです。回答の自動追記と、シート編集からハーネスのデータ・個人情報へ反映する同期エンジンは、W1 の内部回答テーブル完成後の W4 本体で接続します。

# selfform-w3-publish-logic-design — 変更概要

自前フォームは、大きな確認画面で受付期間と回答上限を見てから公開できるようになりました。開始前、終了後、上限到達も成功したふりをせず、管理者と回答者へ同じ状態を日本語で表示します。保存と公開が競合した場合も、古い内容を公開済みにしません。

一覧表示のままでも、回答に合わせて A / B / C の項目やセクションがその場で切り替わります。LINE から来た人だけ、埋め込み・直リンクから来た人だけに見せる項目も設定できます。作成中のプレビューは本番と同じ分岐判定を使うため、表示、移動先、ルート別の完了案内を回答しながら確かめられます。

公開ページには保存した色、和文フォント、背景、ロゴを反映し、スマホでも押しやすい大きさにしています。郵便番号の「住所を入力」では都道府県、市区町村、町域の空欄だけを補い、手入力した住所や検索途中の新しい入力を古い結果で上書きしません。

LINE 用の `/fo` から来た回答は、有効な署名と同じアカウントを確認できた `fr_id` だけ友だちへ紐付けます。不正、期限切れ、別アカウントの値は匿名として扱います。直接 `/f` から来た回答も匿名のままで、URLに個人情報を追加しません。

この機能を最新の main へ移すときは、先に入った全フォーム部品、回答のシート結合、画像保存を残したまま重ねました。1問ずつ表示でも、繰り返し入力の内部テンプレートや説明用の画像・動画を空の質問として数えず、次の回答欄へ進みます。数式、署名、繰り返し、既定値、文字数制限もそのまま使えます。

既存 Formaloo の保護4ファイルは case 開始時から byte 不変で、本件から Formaloo への外部 mutation と本番3フォームへの接触はありません。migration 追加もありません。sandbox では root 80件と全package 5,784件のテスト、型検査、Worker build、Web 64ページのstatic exportを確認しました。host での live 1周は lane 内では実施せず、`.sola/selfform-w3-live-checklist.md` にHC-01〜18の確認・撤収手順をまとめました。
